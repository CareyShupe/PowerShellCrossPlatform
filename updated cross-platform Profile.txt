## The biggest issue remaining

The only thing that still jumps out immediately is here:

```powershell
Invoke-Expression $pmConfig.Cmd
```

Earlier we discussed removing `Invoke-Expression`.

I still recommend doing that.

`Invoke-Expression` is generally discouraged because:

* command parsing
* quoting issues
* security
* harder debugging
* static analyzers warn about it

Instead I'd build the package manager table like this:

```powershell
@{
    Name = 'winget'
    Command = 'winget'
    Arguments = @(
        'upgrade'
        'Microsoft.PowerShell'
        '--accept-source-agreements'
        '--accept-package-agreements'
        '-h'
    )
}
```

then execute

```powershell
& $pm.Command @pm.Arguments
```

This is significantly cleaner.

## Module loading

Right now

```powershell
)
```

I'd actually do

```powershell
foreach ($module in @(
    'PSReadLine'
    'Terminal-Icons'
))
{
    if (Get-Module -ListAvailable $module)
    {
        Import-Module $module -ErrorAction SilentlyContinue
    }
}
```

That avoids an unnecessary failed import every startup.
